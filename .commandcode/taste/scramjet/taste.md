# scramjet
- The Scramjet v2 controller package's RPC-based SW (`controller.sw.js`) has a `postMessage` communication issue where `setupMessagePort` fails because it tries to read `postMessage` on undefined. The controller expects `serviceWorker.postMessage` to be available immediately after registration, but the SW's MessagePort isn't ready yet. Use server-side proxy approach instead of the full controller/RPC architecture. Confidence: 0.85
- When serving proxied HTML through Node.js `fetch()`, strip `content-encoding` and `content-length` headers from the upstream response before forwarding. Node's `fetch()` automatically decompresses the body, so forwarding the original `content-encoding: br` header with decompressed content causes the browser to fail rendering. Also strip `transfer-encoding`, `connection`, `keep-alive`, and `content-security-policy` as hop-by-hop headers. Confidence: 0.85
- The Scramjet v2 runtime (`@mercuryworkshop/scramjet@2.0.67-alpha.1`) sets `self.$scramjet` as a global object. The bundled IIFE (`scramjet_bundled.js`) does NOT expose `$scramjetLoadWorker()` or `$scramjetLoadController()` — those are v1 API patterns. In v2, access `ScramjetServiceWorker` and `ScramjetClient` as properties of `self.$scramjet` (or `globalThis.$scramjet`). The external stub (`scramjet-external.mjs`) re-exports these for ESM consumers. Confidence: 0.80
- The `@mercuryworkshop/scramjet-controller` package's `controller.api.js` expects `globalThis.$scramjet` to already be set (by loading `scramjet.js` or `scramjet_bundled.js` first). Load order matters: runtime bundle before controller bundle. The controller's `Controller` constructor requires `{ config, scramjetConfig, serviceworker, transport }` — the `serviceworker` must be the active ServiceWorker registration, not just `navigator.serviceWorker.controller`. Confidence: 0.75
- The iframe-watch/nested-iframe detection module must skip proxy-prefixed URLs (`/~/sj/`, `/scramjet/`, `/sj-proxy?`, `/s/`) to prevent infinite tab creation loops. When a proxied iframe loads, its internal navigations generate new iframe src values with the proxy prefix — if the iframe-watch treats these as new tabs, it creates exponentially nested iframes. Confidence: 0.85
- For Service Worker files served at root scope (`/sj.sw.js`, `/uv.sw.js`), set `Cache-Control: no-cache, no-store, must-revalidate` to prevent stale SW caching during development. Browsers aggressively cache SW scripts, and without this header, changes to the SW file won't take effect until the 24-hour update check triggers. Confidence: 0.70
- Scramjet v2 runtime ships these dist files: `scramjet.js` (IIFE for pages), `scramjet_bundled.js` (IIFE for all contexts including workers), `scramjet.mjs` (ESM), `scramjet_bundled.mjs` (ESM bundled), `scramjet-external.mjs` (re-exports from globalThis.$scramjet), `scramjet.wasm` (Rust-powered JS rewriter). The `scramjet_bundled.js` is for service workers since it sets `self.$scramjet`. Confidence: 0.75
- The reference Scramjet implementation at `MercuryWorkshop/Scramjet-App` uses a simple standalone SW pattern: `importScripts("/scram/scramjet.all.js")` then `const { ScramjetServiceWorker } = $scramjetLoadWorker()`. However, `scramjet.all.js` does NOT exist in the v2 npm package — the v2 equivalent is `scramjet_bundled.js` and the API changed to `self.$scramjet.ScramjetServiceWorker`. Confidence: 0.80
- Another reference implementation at `Aluminum-Depot/Tinf0il` demonstrates a different Scramjet integration with features including TV, games, chatroom, and live matches — useful for comparison of UI paradigms and feature scope. Confidence: 0.60
- When the v2 controller's `createFrame()` is called with an existing iframe element, the controller wraps it and sets `iframe.src` to the proxied URL using its internal codec. The controller's `wait()` method must be called before creating frames — it initializes the SW communication channel, loads WASM, and syncs cookies from IndexedDB. Confidence: 0.70

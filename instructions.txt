RULES
══════════════════════════════════════════════
1. Verify EVERY change in a real browser (Playwright). "Tests pass" is not
   sufficient. Show the rendered content.
2. Run "node --check <file>" on every JS file before committing. Zero syntax

4. Be honest about what works and what does not.

   and wires the SW. Always run "npm run build" after changing engine code.

   handler wiring. Do NOT load uv.bundle.js separately expecting the SW to see it.

   them before inlining into the classic-script SW. Do NOT importScripts the
   ESM file directly.

══════════════════════════════════════════════
LESSONS LEARNED (avoid these)
══════════════════════════════════════════════
1. SW scope is isolated from the page. importScripts the bundle INSIDE the SW.
2. uv.sw.js defines UVServiceWorker but never instantiates it. Build appends that.
3. getEngine() must NOT cache — settings change the engine at runtime.
4. navigator.serviceWorker.ready ≠ controlling. Wait for controllerchange.
5. bare-mux exports BareMuxConnection as a named ESM export, not window.BareMux.
6. scramjet.bundle.js is ESM (has export). scramjet.all.js is the IIFE build.
   For v2 this may differ — check the package exports field.
7. libcurl-transport ships no wasm. Use Epoxy for both engines.
8. Two root-scoped SWs conflict. Unregister the old one before registering new.
9. filters.js uses ESM export. Strip exports before inlining into the classic SW.
10. jsdom cannot run modules or SWs. Always use Playwright with real Chromium.

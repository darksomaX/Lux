SECTION 1: WHAT LUX IS AND HOW IT WORKS
==================================================
Lux is a self-hostable anti-censorship web proxy. A user runs it on their own machine or server, opens it in a browser, types a blocked URL, and the site loads through an encrypted tunnel that a network censor cannot see.
HOW A PROXIED PAGE LOADS (step by step):
1. The user opens Lux and types "wikipedia.org".
2. Lux's JavaScript registers a service worker (uv.sw.js) at root scope "/". The server serves this worker with a "Service-Worker-Allowed: /" header so it can claim the root scope despite its script living at /uv.sw.js.
3. The service worker's fetch event handler is now installed. Every HTTP request the page makes passes through it.
4. Lux encodes the target URL using UV's XOR codec: wikipedia.org becomes /service/hvtrs8%2F-wkkkpgdka... The encoded string is set as the src of an iframe.
5. The browser tries to fetch /service/hvtrs8%2F-... The service worker intercepts it (it matches the /service/ prefix) and decodes it back to wikipedia.org.
6. The service worker cannot fetch wikipedia.org directly (that is what the censor blocks). Instead, it opens a single WebSocket to the server's /wisp/ endpoint. The wisp protocol multiplexes many logical streams inside that one socket.
7. On top of wisp runs Epoxy, which performs the real HTTPS/TLS exchange with wikipedia.org: the full handshake, the HTTP request, the response.
8. The response flows back: wikipedia.org -> Epoxy -> wisp -> service worker -> iframe. The service worker rewrites all links and resources in the HTML so they also route through /service/...
9. To the censor, the browser made one wss:// connection to your domain. No blocked hostname appeared in any URL, SNI field, or DNS query.
WHY IT EVADES CENSORSHIP:
- IP/hostname blocklists: defeated. The browser only contacts your server.
- URL/SNI string filtering: defeated. The destination is XOR-encoded. Only your hostname is cleartext.
- Logins (Gmail, OAuth): work because all traffic is same-origin to your domain.
- DPI hunting proxy traffic: harder. The tunnel looks like a normal WSS connection.
- Active probing / domain blocking: not defeated. Lux is designed to redeploy fast.
ULTRAVIOLET VS SCRAMJET:
Both are interception proxies by MercuryWorkshop. Ultraviolet uses URL-string rewriting (rewrites every URL to /service/<encoded>, decodes on return). Simple, fast, fully working in Lux. Scramjet uses runtime injection (realm pollution + JS proxies) to intercept property accesses at runtime. More thorough but more complex. Scramjet v1 has a confirmed IndexedDB controller init bug and is deprecated. Scramjet v2 is a complete rewrite, currently alpha, with a separate npm package (@mercuryworkshop/scramjet-controller) and official architecture docs.
SELF-HOSTING:
npm install && npm run build && npm start. Server is Express + wisp-js on one Node process. Behind nginx/Caddy for TLS. Two modes: single (local, no auth) and shared (UUID+password session login, for VPS hosting). Server binds 0.0.0.0 so LAN devices can connect.

SECTION 7: LESSONS LEARNED (avoid these mistakes)
==================================================
1. The UV service worker (uv.sw.js) opens with "var h = self.Ultraviolet" but service workers are ISOLATED from the page. The bundle must be importScripts'd INSIDE the SW, not loaded on the page.
2. uv.sw.js defines the UVServiceWorker class but never instantiates it or attaches a fetch listener. The build must append both.
3. getEngine() must NOT cache. The settings UI changes the engine at runtime. A stale cache silently keeps the old engine.
4. navigator.serviceWorker.ready resolves when a SW is "active" but NOT when it "controls" the page. You must wait for navigator.serviceWorker.controller to be your SW before navigating, using the controllerchange event.
5. bare-mux exports BareMuxConnection as a named ESM export, NOT as window.BareMux. transport.js attaches it to window.BareMux for compatibility.
6. scramjet.bundle.js is an ESM build (has export statements). Loading it as a classic script throws "Unexpected token export". scramjet.all.js is the IIFE build. For v2 this may change.
7. The libcurl-transport npm package does NOT ship its wasm. Epoxy's wasm IS shipped. Both engines should use Epoxy.
8. Two root-scoped service workers cannot coexist. Switching engines must unregister the old one first, but carefully: do not unregister an installing SW (it might be yourself).
9. Node fetch() does not let you override the Host header. Use raw TCP (node:net) to test censor middleware with custom Host headers.
10. jsdom cannot run type="module" scripts or service workers. It gives false failures. Always use Playwright with real Chromium for feature verification.
